# Game QA Portfolio — Example of Feature Test Cases & System Test Specifications

These are sample QA artifacts written for portfolio purposes.
They are not meant to look like Jira tickets copied from a production project. They are written as if I were preparing test coverage for a feature review with QA, design and development.

The focus is not only “does it work?”, but also:

* does the feature preserve player trust?
* does it communicate state clearly?
* does it survive edge cases?
* does it interact safely with other systems?
* does the result match the intended player experience?
